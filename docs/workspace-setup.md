# Workspace Setup

Bootstrapping splits into two layers:

- **Global (once per machine)** â€” agent runtimes, their home-directory
  config, and credentials in the OS keychain. Shared across *every*
  workspace on the machine; never lives in the repo.
- **Workspace-local (once per clone)** â€” files inside this repo. Tracked
  files come with the clone; the `*.example` / `*.local` files are copied
  and tailored per user.

Do the global layer first; the workspace layer depends on it (e.g. the MCP
token export and the `gh` login).

## Global â€” once per machine

Authoritative list: `docs/workspace-structure.md` â†’ "User-Level Files
(Outside the Workspace)". These all live under your home directory and are
**never** checked in.

| Item | Where | Used for | Verify |
|---|---|---|---|
| Agent CLIs | `claude`, `codex`, `opencode`, `gemini` on `PATH` | The agent runtimes themselves | `claude --version` |
| `gh` CLI + login | `~/.config/gh/`, system keychain | GitHub auth; source of the MCP token | `gh auth status` |
| MCP token export | `~/.zshrc` (or per-shell) | Expands `${GITHUB_PERSONAL_ACCESS_TOKEN}` for MCP servers | `echo "${GITHUB_PERSONAL_ACCESS_TOKEN:+set}"` |
| Codex MCP registration | `~/.codex/config.toml` | GitHub MCP for Codex | `codex` MCP list |
| Gemini MCP registration | `~/.gemini/settings.json` | GitHub MCP for Gemini | restart Gemini, check tools |
| MCP wrapper scripts (if used) | `~/.mcp-scripts/` | Launch scripts referenced by MCP config | run the script |
| Service credentials | OS keychain (`security â€¦`), `~/.pgpass`, `~/.aws/`, â€¦ | Tokens/passwords for external services | per-service verify cmd in `docs/service-access.md` |

Optional agent toolchain (status line, superpowers plugin, Matt Pocock
skills, graphify) is set up at this global layer too â€” see
`docs/recommended-tooling.md` for install steps and the per-repo bits.

Notes:
- **Credentials never leave the keychain.** `docs/service-access.md` lists
  the retrieve/verify command per service; `.env` holds only non-secret
  identifiers.
- Per-runtime MCP config details (exact TOML/JSON, remote vs. Docker server)
  are in `docs/mcp-setup.md` â€” that doc is the source of truth for the
  Codex/Gemini home-directory snippets above.
- Agent *memory* and *personal preferences* belong in the runtime's own
  store (e.g. `~/.claude/projects/<id>/memory/`), **not** in this repo.

## Workspace-local â€” once per clone

Authoritative list: `docs/workspace-structure.md` â†’ "Gitignored vs. Checked
In". Tracked files arrive with the clone; you create the local/secret ones.

```bash
# 1. Clone
git clone <clone-url> <workspace-dir> && cd <workspace-dir>

# 2. Bootstrap: creates .env, the entrypoint symlinks, repos/README.md,
#    and (with --clone-repos) clones product repos from the registry.
./scripts/setup.sh            # add --clone-repos for a multi-repo workspace

# 3. Per-user config copies (all gitignored â€” tailor to your machine):
cp .env.example .env                              # set non-secret identifiers
cp .mcp.json.example .mcp.json                    # Claude Code project MCP
cp .claude/settings.json.example .claude/settings.local.json   # CC permissions

# 4. Preflight credentials and verify structure
./scripts/check-service-access.sh
./scripts/check-workspace-structure.sh
```

What each local file is for:

| File | Tracked? | Action on a fresh clone |
|---|---|---|
| `CONTEXT.md` + `CLAUDE.md`/`AGENTS.md`/`GEMINI.md` symlinks | yes | nothing â€” read it |
| `opencode.json`, `.gemini/settings.json` | yes | nothing â€” OpenCode/Gemini pick them up |
| `.env.example` â†’ `.env` | example tracked | copy, set identifiers (no secrets) |
| `.mcp.json.example` â†’ `.mcp.json` | example tracked | copy for Claude Code |
| `.claude/settings.json.example` â†’ `.claude/settings.local.json` | example tracked | copy, tailor permissions |
| `repos/*`, `references/*` | no (clones) | populated by `setup.sh --clone-repos` |
| `.service-access.local.json` | no | generated by `check-service-access.sh` |

> `scripts/setup.sh` is idempotent â€” re-run it any time to repair missing
> symlinks or config copies. This also fixes the case where the
> `CLAUDE.md`/`AGENTS.md`/`GEMINI.md`/`repos/README.md` symlinks were
> **flattened into plain files** (Windows clones without `core.symlinks`, or
> some copy-based "Use this template" flows): delete the flattened files and
> re-run `setup.sh`, or `git config core.symlinks true` before cloning.

## Quick verification

```bash
gh auth status                                    # global: GitHub auth
echo "${GITHUB_PERSONAL_ACCESS_TOKEN:+token set}" # global: MCP token exported
claude mcp list                                   # local:  GitHub MCP visible to Claude Code
./scripts/check-workspace-structure.sh            # local:  symlinks resolve, dirs present
```
