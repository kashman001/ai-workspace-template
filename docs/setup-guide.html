01 · What you get

The template is a coordination layer — not product code. It gives every agent and human a single known place to find context, conventions, and active work. Everything project-specific is a <…> placeholder or a stub with a header comment, so you fill in, you don't reverse-engineer.

Figure 1 — Top-level layout. Full detail in docs/workspace-structure.md

02 · The mental model

One file is the front door. The three agent entrypoints are symlinks to it, so every runtime reads the exact same context — edit once, all agents see it.

Figure 2 — Claude Code, Codex (AGENTS.md), and Gemini all resolve to one file

03 · Quick start

Five steps from clone to a working workspace. Steps 1–3 are mechanical; step 4 is where you make it your project.

Figure 3 — The setup pipeline
1

Create your workspace from the template

On GitHub, click “Use this template”, or clone and reset git history:

git clone https://github.com/<you>/ai-workspace-template.git <project>-workspace
cd <project>-workspace
rm -rf .git && git init
2

Bootstrap the local files

Run the setup script (or do the manual equivalents — it ships as an executable stub). Add --clone-repos for a multi-repo workspace.

./scripts/setup.sh                 # creates .env, symlinks, repos/README.md
./scripts/check-workspace-structure.sh   # verify symlinks resolve, dirs present
3

Wire up agents & MCP

Copy the per-user config templates (all gitignored) and export the GitHub MCP token. Full per-runtime detail lives in docs/mcp-setup.md.

cp .env.example .env
cp .mcp.json.example .mcp.json
cp .claude/settings.json.example .claude/settings.local.json
export GITHUB_PERSONAL_ACCESS_TOKEN="$(gh auth token)"
4

Make it your project

Replace every <…> placeholder and resolve the TODO / “Fill in” comments — start with CONTEXT.md (it loads into every agent session). Find what's left:

grep -rIn --exclude-dir=.git -e '<[a-z-]\+>' -e 'TODO' -e 'Fill in:' .
  • CONTEXT.md — name, purpose, single- vs multi-repo model
  • docs/repos-registry.md — one entry per product repo
  • docs/service-access.md — your GitHub username; add a section per service
  • SPEC.md — product intent (or delete for a pure coordination layer)
5

Start building

Decide on the optional bits: graphify is wired by default (.gemini/settings.json, .opencode/plugins/graphify.js) — delete it if unused. Add reusable workflows under skills/ and list them in CONTEXT.md.

Deeper reference: docs/template-usage.md · docs/workspace-setup.md · docs/workspace-structure.md

04 · Global vs. workspace-local setup

Setup splits into two layers. Do the GLOBAL layer once per machine; the LOCAL layer once per clone. The local layer depends on the global one (the MCP token and the gh login).

06 · Recommended toolchain

Optional external tools this workflow assumes. They're global / per-user — installed once per machine, never committed to the repo — except for two that also need a per-repo step. Full setup in docs/recommended-tooling.md.

Figure 5 — Global install vs. per-repo setup. Detail in docs/recommended-tooling.md

Quick reference:

# status line
npx -y ccstatusline@latest            # editor; wire statusLine in ~/.claude/settings.json

# plugins (official marketplace is built in)

07 · For AI agents

If you are an AI agent setting this up

  • Read CONTEXT.md first — it's the front door and loads the project's conventions and the “Agent Coding Principles” / “Agent Context Discipline” rules.
  • Treat docs/workspace-structure.md as the spec. Its “Agent Bootstrap Instructions” section is a step-by-step procedure for scaffolding or extending a workspace — files to create, stub-header format, and post-scaffold verification checks.
  • Confirm preconditions before creating files — workspace name, repo list, editors/runtimes, and external services. Don't invent repo names or clone URLs.
  • Persist intermediate state to work/<username>_<project>/ so it survives context compaction — not to conversation memory.
  • Disk is the source of truth. Re-read files before asserting facts; verify symlinks resolve and scripts are executable after scaffolding.

Verify before you call it done

gh auth status                                    # global: GitHub auth
echo "${GITHUB_PERSONAL_ACCESS_TOKEN:+token set}" # global: MCP token exported
claude mcp list                                   # local:  GitHub MCP visible
./scripts/check-workspace-structure.sh            # local:  symlinks + dirs OK
