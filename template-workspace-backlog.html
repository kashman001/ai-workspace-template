Maintaining this backlog

Treat this file as a lightweight issue tracker for the template itself. The rules:

  1. IDs are stable and severity-scoped. High, Medium, Low, Decision + a number (e.g. M6). Never renumber or reuse an ID — once assigned it's permanent, even after the item is resolved.
  2. Every finding carries a status badge: Open · Resolved · Decided · Won't fix.
  3. When you fix an item: flip its badge to Resolved, add a Fixed: line describing the change (and the commit/PR if handy), and add class="resolved" to its <div class="find …">.
  4. When you find a new issue: append a new finding card under the right severity heading with the next free ID, status Open, and the Evidence / Impact / Fix lines.
  5. Update the scorecard counts and the "Last updated" date in the header on every change.
  6. Commit convention: reference the ID, e.g. git commit -m "Fix M6: …", so the git log and this file stay in sync.
  7. Re-review trigger: after any structural change to the template (new config file, new doc, new agent runtime), do a fresh evidence pass and append what it surfaces.

Tip: the matching skill for turning these into real tracked issues is to-issues (see docs/recommended-tooling.md) if you'd rather move them to GitHub Issues.

High

H1 Bootstrap scripts were exit 1 stubs while the Quick Start told you to run them HighResolvedscripts/

Evidence — the three scripts/*.sh only echoed a TODO and exited 1, yet setup-guide.html Step 2 and workspace-setup.md instructed running them.

Fixed: implemented real, idempotent setup.sh (symlinks, .env/ .mcp.json/.claude/settings.local.json copies, --clone-repos that parses the registry), check-workspace-structure.sh (dirs/symlinks/exec/gitignore checks, non-zero on failure), and check-service-access.sh (gh + token preflight, writes the cache). Guide/setup wording updated.

H2 Gemini graphify hook called python (not python3) HighResolved.gemini/settings.json

Evidence — the BeforeTool hook ran python -c "…", which fails on python3-only machines, erroring on every Gemini file read/list.

Fixed: rewrote the hook as a pure POSIX-shell one-liner (if [ -f graphify-out/graph.json ]; then printf …) that emits the allow/context JSON with no Python dependency at all.

Decisions

D1 Four coding principles duplicated: CONTEXT.md vs global.md DecisionDecidedCONTEXT.md

Decided: keep the principles in CONTEXT.md. A public template can't assume the agent-context global.md exists, so self-containment wins over DRY. The intentional duplication is documented in recommended-tooling.md.

D2 init-project-ai-infra redundancy for template-cloned workspaces DecisionDecidedrecommended-tooling.md

Decided: still recommend running init-project-ai-infra --full — not to create the symlinks (the template ships those) but to scaffold docs/adr/ + docs/agents/ that the Matt Pocock skills expect; it's idempotent and won't clobber existing files. Docs updated.
